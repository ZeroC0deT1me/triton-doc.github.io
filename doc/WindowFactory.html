WindowFactory

Package triton.ui Modifier public final — non-instantiable Inner PendingAnimationData · Hit

All-static factory for custom Swing window chrome. Builds a macOS-style traffic-light title bar (red/yellow/green + purple Move-to-Screen), installs an 8-px edge-and-corner resizer via a global AWTEventListener, manages a per-class animation queue, and provides lightweight dialog chrome helpers.

create()

Primary factory entry point. Creates an undecorated JFrame, applies chrome, enrolls with EchelonWindowPlacer, binds float-above-desktop, and — if an animation is pending — sets opacity to 0.0f and wires a WindowAdapter that fires the animation 100 ms after windowOpened (to let the placer settle first).

// Basic
JFrame f = WindowFactory.create("My Window");
f.setSize(800, 600);
f.setVisible(true);

// With animation — call setPendingAnimation first
WindowFactory.setPendingAnimation("zoom-reveal", sourceButton);
JFrame f = WindowFactory.create("My Window");
Backward-compatible overload create(String title, boolean unusedCustomEverywhere) accepts but ignores the boolean — it exists for call sites that passed a flag before the factory was simplified.

Float-above-desktop is enabled by default. It can be suppressed:

ScopeMechanism
System-wideJVM property echelon.floatAboveDesktopOnly=false
Per-windowRoot pane client property "echelon.floatAboveDesktopOnly" = Boolean.FALSE

applyCustomChrome()

Can be called on any pre-existing undecorated JFrame to retrofit Triton chrome. Forces setUndecorated(true) if needed, replaces the content pane with a new ChromeContentPane, adds the title bar to NORTH, installs the resizer and panic-unstick, then marks the root pane with "echelon.chromed" = Boolean.TRUE to prevent double-wrapping.

buildTitleBar()

Private method that returns a JPanel named "EchelonTitleBar" — this name is how ChromeContentPane and EchelonChromeBridge detect existing chrome. Layout is BorderLayout:

RegionContent
WESTTraffic-light dots panel (red, yellow, green, purple)
CENTERRight-aligned bold title JLabel, live-updated via frame's propertyChangeListener("title")
EASTReserved JPanel tagged "echelon.titlebar.rightBox"=true for module extras

Background color comes from UIManager.getColor("Panel.background") with a null fallback to #F2F2F2.

Traffic Lights

Four 14×14 opaque circular buttons, created by the private macDot(JFrame, Color, ActionListener) helper:

Red #FF5F57 — dispatches WINDOW_CLOSING event
Yellow #FEBC2E — calls frame.setState(Frame.ICONIFIED)
Green #28C840 — calls private toggleMax()
Purple #B388FF — calls showMoveToScreenDialog(frame); tooltip "Move to Screen"

Right Box Slot

An empty FlowLayout(TRAILING) panel in BorderLayout.EAST of the title bar, tagged with client property "echelon.titlebar.rightBox" = Boolean.TRUE. Modules can retrieve and populate it to add toolbar buttons inside the title bar without touching chrome internals.

Drag & Double-Click

A shared MouseAdapter is added to both the bar panel and the title label. On mousePressed it records the offset of the cursor from the frame's top-left corner. On mouseDragged it moves the frame. On double-click (BUTTON1, count 2) it calls toggleMax().

Cursor interference fix The bar itself does NOT set MOVE_CURSOR on itself (this was removed). Setting a cursor on the bar would override the resizer's edge cursors when the mouse is near the top border.

installResizer()

Registers a global AWTEventListener on MOUSE_EVENT_MASK | MOUSE_MOTION_EVENT_MASK. The listener is scoped to a specific frame by checking SwingUtilities.getWindowAncestor(src) == f and is unregistered on windowClosed.

PropertyValue / notes
Hit border8 px from each edge (increased from 6 for easier grabbing)

applyDialogChrome()

Minimal chrome for JDialog. Injects a title bar with a single red close dot (→ dlg.dispose()), a centered bold title label, and a drag adapter. Wraps the existing content pane in a new JPanel rather than replacing it. No resizer is installed on dialogs.

installPanicUnstick()

Binds ESC (in WHEN_IN_FOCUSED_WINDOW scope) to an action that iterates all open Window instances, resets their cursors to default, hides and clears all glass panes via makeGlassPaneSafe(), and clears global keyboard focus. Recovers the UI from any stuck drag or resize state.

// makeGlassPaneSafe — also public for external callers
WindowFactory.makeGlassPaneSafe(frame); // hides glass pane, clears children

showMoveToScreenDialog()

Tries EnhancedMoveToScreenPanel.show(frame) first (carousel UI with snap grid). On any failure falls back to a plain JOptionPane.showInputDialog listing each screen as "Screen N (WxH @ x,y)". After selection, moveFrameToBounds() centers the frame on the target rectangle and calls EchelonWindowPlacer.manage(frame) again.

Animation Queue

Static ConcurrentLinkedQueue<PendingAnimationData>. Call setPendingAnimation(type, source) immediately before create(). Expired entries (older than 5 s) are drained from the head on each create() call before polling the next valid entry.

WindowFactory.setPendingAnimation("zoom-reveal", clickedButton);
JFrame f = WindowFactory.create("Module");
f.setVisible(true); // animation fires ~100ms after windowOpened

PendingAnimationData

Fields & Expiry

Mirrors EchelonFrame.PendingAnimation — same structure, separate queue, separate consumer (create() vs windowOpened).

private static class PendingAnimationData {
    final String    type;       // "zoom-reveal" | "fade" | "none"
    final Component source;     // button origin for zoom
    final long      timestamp;

    boolean isExpired() {
        return (System.currentTimeMillis() - timestamp) > 5000;
    }
}
Type "none" Passing "none" evaluates to hasAnimation = false in create() — the frame is not hidden and no delegate runs. Use this to consume a queue slot without triggering any effect.
